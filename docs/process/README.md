# docs/process/ — プロセス定義

開発フローやワークフローのうち、**プロジェクト固有の** 取り決めをまとめる。

## docs/process/ と .claude/rules/ の違い

| 場所             | 内容                                     | 例                                   |
| ---------------- | ---------------------------------------- | ------------------------------------ |
| `.claude/rules/` | エージェントが自動ロードするコード規約   | Kotlin規約, Compose規約, 依存方向    |
| `docs/process/`  | 人間が参照する作業手順・運用ルール       | リリース手順, インシデント対応, 命名 |

`rules` は**機械が常に参照する規範**、`process` は**人間が手順として読む文書**、という分担。

## 配置するもの

- **リリース手順** — どのブランチで、何をビルドし、Play Store にどう上げるか
- **インシデント対応** — クラッシュ報告を受けた時の調査フロー
- **PR運用** — レビュー観点、マージ条件、Squash の方針
- **障害台帳の運用** — GitHub Issue ラベル設計、トリアージのリズム
- **オフボーディング/オンボーディング** — 開発に参加する人向けのセットアップ

## 推奨ファイル (必要になったら追加)

```
docs/process/
├── README.md
├── release.md         # リリース手順
├── incident.md        # インシデント対応
├── pr-review.md       # PR レビュー観点
└── triage.md          # Issue トリアージ
```

最初は空でよい。実際に運用が固まったら言語化して追加する。
