# 仕様: ゲームフロー（出題・回答・正誤照会）

## 概要

定石を選択し、クイズ形式で最善手を練習する一連のゲームフローを定義する。
トップ画面で定石を選択し、出題・回答・正誤照会のサイクルを繰り返す。

## 用語定義

- **定石**: オセロ序盤の最善手順に名前が付いたもの（例: 虎定石、うさぎ定石）
- **出題局面**: 定石の進行上の局面。最善進行または最善から評価値が若干低い進行の局面
- **評価値**: Edax が計算する着手の強さ（数値）
- **最善手**: 当該局面で最も評価値が高い手
- **正解手**: 最善手（出題の答え）

## ユースケース / シナリオ

### UC-01: 定石を選んで練習を開始する
**Given** アプリ起動後、トップ画面が表示されている  
**When** ユーザーが定石一覧から定石を1つ選択する  
**Then** 出題画面に遷移し、選択した定石の局面が出題される

### UC-02: 局面に手を打つ
**Given** 出題画面が表示されている  
**When** ユーザーがオセロ盤上の打てるマスをタップする  
**Then** 石を置いて裏返し（アニメーションあり）、正誤照会を表示する

### UC-03: 打てないマスをタップする
**Given** 出題画面が表示されている  
**When** ユーザーがルール上打てないマスをタップする  
**Then** 「そのマスには打てません」ダイアログを表示し、有効な場所に打つよう促す

### UC-04: 正誤照会を確認する
**Given** 回答済みの状態で正誤照会が表示されている  
**When** ユーザーが画面を確認する  
**Then** 回答手の正解/不正解と、全着手可能な手の評価値を表示する

### UC-05: 次の問題に進む / トップに戻る
**Given** 正誤照会画面が表示されている  
**When** ユーザーが「次の問題」または「トップに戻る」を選択する  
**Then** 次の出題またはトップ画面に遷移する

## ルール / 振る舞い

### 出題

- トップ画面（定石選択画面）で定石一覧を表示し、ユーザーに選択させる
- 定石選択後、出題画面に遷移して繰り返し出題する
- 出題局面: 最善の進行および最善でない手を打った局面を含む
- 出題画面には「最初の画面に戻る」ボタンを表示する

### 再出題制御

- 同一局面に 3 回正解したら、以降はその局面を出題しない
- 正解回数は定石ごとに分類して保持する

### 回答

- 打てるマスをタップすることで回答する
- 打てないマスをタップした場合はダイアログを出し、回答はキャンセルされる
- 有効な手のマスをタップしたときのみ石を裏返す
- 石を裏返す際はアニメーションを入れる

### 正誤照会（回答後）

- 回答手が正解か不正解かを表示する
- 着手可能な全手の評価値を表示する
- 評価値の色分け:
  - 正解手: 水色
  - 不正解手（回答した手）: 赤色
  - 回答手以外 — 回答手と同じかそれ以上の評価値: 緑色
  - 回答手以外 — 回答手より評価値が低い: オレンジ色

### 回答後の画面遷移

- 正誤照会画面内で「次の出題」または「トップ画面に戻る」をボタンで選択できる

## 制約・前提

- 対象は序盤定石のみ。中盤・終盤戦術は扱わない
- 評価値の算出には Edax API を使用する（詳細は設計ドキュメント参照）

## スコープ外

- 通信対戦
- 中盤・終盤の手の評価・練習
- ユーザーアカウント・ランキング機能
