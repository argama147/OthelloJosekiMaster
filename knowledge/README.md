# knowledge/ — ドメイン知識集

オセロ・定石・UX検討といった**「コードや仕様の前提となるドメイン知識」**を Markdown でまとめる場所。

仕様 (`docs/spec/`) や設計 (`docs/design/`) と異なり、ここは「前提として参照する事実」を集める。

## 配置するもの

- **オセロのルール** — 公式ルール、置換禁止条件、ゲーム終了条件、得点計算
- **定石 (joseki) の体系** — 主要定石の分類、進行手順、評価
- **オセロ用語辞典** — 開放度、確定石、X打ち等の専門用語
- **UI/UX 参考資料** — 既存のオセロアプリのUIパターン、Material 3 ガイドライン参照
- **データソース・出典** — 定石データの根拠 (書籍・論文・公開データセット)

## 配置しないもの

- ❌ 仕様 (この機能は何をする) → `docs/spec/`
- ❌ 設計 (どう実装するか) → `docs/design/`
- ❌ 設計判断の経緯 → `docs/adr/`
- ❌ 作業の進行状態 → `.claude/state/`

## ファイル例 (推奨)

```
knowledge/
├── README.md                    # このファイル
├── othello-rules.md             # 公式ルール
├── joseki-glossary.md           # 定石の用語辞典
├── joseki-classification.md     # 定石体系 (虎・牛・うさぎ等)
├── ux-references.md             # 参考UI/競合アプリ
└── data-sources.md              # 定石データの出典
```

最初は空でよい。仕様検討中に「これは前提知識として後から参照したい」と感じたら随時追加していく。
